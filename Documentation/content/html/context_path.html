The "path" Context

The "path" Context of dashR is included for handling file paths. It automatically addes \ characters where needed.

Included Methods

dashR("path").combine (arr[spec1, spec2, spec3]);
  • The parameter arr is an array of string(s) to concatenate into a path.
Example use of .combine 

    var folder = "c:\\mystuff";
    var subFolder = "mydocuments"
    var fileName = "mine.txt";

    // load my obj or return an empty one.
    var fileSpec = dashR("path").combine([folder, subFolder, fileName]);

    console.log(fileSpec);
    c:\\mystuff\\mydocuments\\mine.txt
dashR("path").relative (from, to);
  • This functions returns a new path that is relative from the from folder.
Example use of .relative 

    var folder = "c:\\Dev\\Apps\\Basic";    
    var fileSpec = dashR("path").relative(folder, "..\\..\\Testing\\file.txt");

    console.log(fileSpec);
    c:\Dev\Testinbg\file.txt
dashR("path").map.for (alias[required], fileSpec[optional]);
  • Map is an object for saving file specs (paths/fileNames) to reference by alias.
  • The alias is the file specs handle to reference it by
  • The fileSpec is the full path and filename for the file (including the file extension).
  • If only the alias is provided it returns the filespec, if both are porvided it saves it.
  • The App Object containts one of these objects built in with the name fMap. See Below
Example use of .map 

    var fMap = new dashR("path").map();
    fMap.for("undo", "N:\\storage1\\rename_undo.json");    

    var undoFileSpec = fMap.for("undo");
    // undoFileSpec = N:\\storage1\\rename_undo.json

    dashR("file").delete(fMap.for("undo"));

    // 
    var dashRApp = dashR().newApp({
        ...
    })

    dashRApp.fMap("text", "test.txt");
    var fText = fMap.spec("text"); 
    // should be test.txt

    dashRApp.fMap("text", "C:\\desktop\\test.txt");
    var fText = fMap.for("text"); 
    // should be c:\\Desktop\\test.txt
