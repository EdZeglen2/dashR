The "file" Context

The "file" Context of dashR is included for handling basic file operations. Load, save, remame, copy, delete, etc...

Included Methods

dashR("file").fso;
  • .fso returns a Windows Scripting File System Object
dashR("file").exists (fileSpec);
  • The fileSpec to test for exists on the computer.
dashR("file").get (fileSpec);
  • The fileSpec is the file to load on the computer .
  • The Windows File Object properties include:
    .Attributes
    .DateCreated
    .DateLastAccessed
    .DateLastModified
    .Drive
    .Name
    .ParentFolder
    .Path
    .ShortName
    .ShortPath
    .Size
    .Type
dashR("file").name (fileSpec);
  • This returns just the files name, no path or file extension.
  • The fileSpec is the path to the file.
dashR("file").ext (fileSpec);
  • This returns just the files extension, without a period.
  • The fileSpec is the path to the file.
dashR("file").load (fileSpec, into[optional], separator[optional]);
  • This returns either the contents of the file (usually text) or an array of lines split on the seperator
  • The fileSpec is the path to the file.
  • The parameter into is an optional array to fill.
  • The parameter separator is an optional character to split the file data on.
  • This is mostly just used to load text files
  • One potential use may be to read a file line by line, use the crlf (\r\n) to split the file
    into an array and process each line
dashR("file").save (fileSpec, text);
  • The fileSpec is the path to the file to save to.
  • The parameter text is the text to put in the file.
dashR("file").delete (fileSpec);
  • The fileSpec is the path to the file to delete.
dashR("file").copy (source, destination, overwrite[optional]);
  • The parameter source is the file to copy.
  • The parameter destination is a folder to copy the file to.
  • The optional boolean parameter overwrite specifies if the file
    should be overwritten, if not provided it assumes true.
dashR("file").copy_to (targetFolder, sourceSpec);
  • This function shows the Windows copy dialog window with a progress bar.
  • While the copy is in process the application will halt and not respond until it finishes.
  • The parameter targetFolder is where to copy to.
  • The parameter sourceSpec is what to copy (can be a file or folder).
dashR("file").rename (fileSpec, newfilename);
  • The parameter fileSpec is the file to be renamed.
  • The parameter newfilename is the new name of the file.
dashR("file").download (url, fileSpec, fnDone);
  • The parameter url is the url of a file or image to download.
  • The parameter fileSpec is file location to save the downloaded file to.
  • The parameter fnDone is a function to call when the asyncronous operation is finished.
