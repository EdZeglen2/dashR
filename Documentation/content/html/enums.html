dashR Enums

dashR Enums add to the window global scope and are available from any window JavaScript code (including dashR). Simply add an enums array to your Context to define your custom Enums.

Example use of Enums in a Context 
    dashR().addContext({
        name: "color-picker",
        enums: [
            { 
                name: "Color", 
                values: {
                    RED: 0,
                    GREEN: 1,
                    YELLOW: 2
                }
            }
        ],
        js: {
            index: 0,
            init: function () {
                // default the index to green.
                index = Color.GREEN;
            },
            picked: function (color)  {
                if (color == COLOR.RED) {
                    // picked RED
                }
            }
        }
    })
Summary

  • Enums, like JavaScript are case sensitive, Color.RED and color.red are not the same.
  • Enums are not specific to the context that added them.
  • Enums exist in the .window global space and can be used from any functions or other Contexts
  • You should never use a name that already exists, instead, use dashes or underscore to keep things unique to your application.
