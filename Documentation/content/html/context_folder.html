The "folder" Context

The "folder" Context of dashR is included for handling basic folder operations. brwose, create, remame, copy, delete, etc...

Included Methods

dashR("folder").fso;
  • .fso returns a Windows Scripting File System Object
dashR("folder").exists (folderSpec);
  • The folderSpec to test for exists on the computer.
dashR("folder").get (folderSpec);
  • The folderSpec is the folder to load on the computer .
  • The Windows Folder Object properties include:
    .Name
    .Size
    .Type
dashR("folder").parent (folderSpec);
  • This returns just the folders parent folderSpec.
  • The folderSpec is the path to the folder to want the parent of.
dashR("folder").sub_folders (folderSpec, into[optional], separator[optional]);
  • This returns just the folders in the folder (no files).
  • The folderSpec is the path to the folder.
  • The into parameter is an array to fill with folder objects.
  • file object properties inculde
    .name
    .spec
dashR("folder").files (folderSpec, into);
  • This returns just the files in the folder (no folders).
  • The folderSpec is the path to the folder.
  • The into parameter is an array to fill with file objects.
  • file object properties inculde
    .name
    .spec
    .size
    .ext
    .mDate
dashR("folder").traverse (path, into);
  • This populates an array with all the folder and file information in the tree including subfolders
  • The path is the path to the folder to start with to.
  • The parameter into is an array to put the data.
dashR("folder").create (folderSpec);
  • The folderSpec is the path to the folder to create.
  • This will fail if the folder already exists, see .make below or use .exists above, first to check before calling
dashR("folder").make (folderSpec);
  • This will create a folder, but only if the folder does not exist.
  • The parameter folderSpec is the folder to make sure it exists.
dashR("folder").delete (folderSpec);
  • The parameter folderSpec is the folder to delete.
dashR("folder").copy (source, destination, overwrite);
  • The parameter source is the folder to be copied.
  • The parameter destination is the folder to copy source to.
  • The parameter overwrite specifies to overwrite the folder and everything in it.
    If unspecified, assumes true.
dashR("folder").copy_to (targetFolder, sourceSpec);
  • This function shows the Windows move dialog window with a progress bar.
  • The parameter source is the folder to have its contents copied.
  • The parameter destination is the folder to copy source items to.
dashR("folder").move_to (toMove, moveTo);
  • This function shows the Windows move dialog window with a progress bar.
  • The parameter toMove is a file to move.
  • The parameter toMove may be a folder to move (? never tried it).
  • The parameter moveTo is the folder to move toMove to.
dashR("folder").browse (startinfolder, dialogtitle);
  • Brings up the windows select a folder dialog box.
  • If the user cancels the dialog null is returned. Check for this.
  • The parameter startinfolder is folder to start the dialog in.
  • The parameter dialogtitle is the dialogs title bar message.
