dashR Elements

The dashR Library will scan for its own HTML tags in the document on app.load(). If found, it will attempt to locate the Context and create the item type matching the type attribute of the dashR tag. Class and all other attributes are optional but can style your element. Make sure to include any .css files your element may need in the head of the page.


Requirements for usage

The dashR library expects all dashR elements to have a unique id, a unique name, a context reference (by name) and a type.

Example HTML 
<dashR 
    id="selector1" class="custom-selector" 
    name="selector1" context="icons" type="selector">
    (Loading...)
</dashR>

Note in the example above we have provided all four of the required attributes to the tag. In this example dashR will attempt to create the "selector" object inside the "icons" context. Below is the code for adding this context to dashR.

Example JavaScript 
    dashR().addContext({
        name: "icons",
        js: {
            selector: function () {
                return {
                    items: [],
                    selected: null,
                    index: -1,
                    init: function () {
                        ...
                    },
                    bind: function () {
                        ...
                    }
                }
            }
        }
    });

The above example does not reflect a real object and you would have more methods and properties than we have shown.
